摩擦点：测试覆盖率为零

整个项目没有任何测试文件。两个平台的 wrapper 脚本有大量边界行为（JSON 解析、进程管理、PTY fallback、错误处理），但零测试覆盖。

git 历史显示多次修复都是靠人工发现：
- agent_message length filter 反复调整（f266fd8, 244adef, 25d9933）
- shell filter regex 修正（25d9933, 80d975e）
- PTY allocation 兼容性问题（0b9f1d0, 051523c）

这些问题如果有测试本来可以避免。当前的开发流程是"用户报 bug → 修复 → 提交"，缺少预防机制。
